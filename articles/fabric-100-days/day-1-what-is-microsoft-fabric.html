What is Microsoft Fabric?

Microsoft Fabric is a unified, end-to-end analytics platform released by Microsoft in 2023. It brings together data engineering, data integration, data science, real-time analytics, and business intelligence under a single SaaS product — all running on top of a shared storage layer called OneLake.

Before Fabric, organisations had to stitch together Azure Synapse, Azure Data Factory, Azure Databricks, Power BI Premium, and several other services. Each had its own security model, billing meter, storage layer, and developer experience. Fabric collapses all of that into one coherent platform.

Fabric is organised into six experience areas, each targeting a specific persona:

OneLake — The Foundation of Everything

OneLake is Fabric's unified storage layer — think of it as OneDrive for analytics data. Every workspace in your tenant gets its own section of OneLake, and all Fabric workloads read from and write to the same underlying storage without copying data.

  • Built on Azure Data Lake Storage Gen2 (ADLS Gen2)
  • Data is stored in open formats: Delta Parquet for tables, raw files for landing zones
  • A single namespace across the entire tenant — no per-service silos
  • Shortcut feature lets you mount external data (ADLS Gen2, AWS S3, Google Cloud Storage) without moving it

Key Architectural Concepts

Lakehouse

A Lakehouse in Fabric combines the flexibility of a data lake with the query performance of a data warehouse. It stores Delta tables in OneLake and exposes them automatically via a SQL Analytics Endpoint — so you can query them with T-SQL without any configuration.

Workspaces

A Workspace is the organisational unit in Fabric. It maps to a capacity, has its own role-based access control (RBAC), and contains items like Lakehouses, Warehouses, Notebooks, Pipelines, Semantic Models, and Reports.

Capacity

All compute in Fabric is backed by a Capacity (F-SKU or P-SKU). A capacity provides a pool of Capacity Units (CUs) that are shared across all workloads in workspaces assigned to it. You can pause a capacity when not in use to save cost.

Why It Matters for Data Teams

  1. Single identity & security model — One Azure AD (Entra ID) identity works across all workloads. No separate service principals per tool.
  2. No data duplication — Because every workload reads from OneLake, a Spark notebook and a Power BI report can share the same Delta table without an ETL copy step.
  3. Unified billing — One capacity meter (CUs) covers all workloads. No per-query DWU billing for the warehouse, no separate streaming ASU billing.
  4. Built-in governance — Microsoft Purview is deeply integrated for data cataloguing, lineage, and sensitivity labels.
  5. Open formats — Delta Parquet is open source. You are never locked into a proprietary storage format.

Getting Started — Quick Checklist

  • ☑ Enable Fabric in your Microsoft 365 / Azure tenant admin portal
  • ☑ Create a Fabric capacity (F2 SKU is the minimum for most features; F64 for full enterprise)
  • ☑ Create a Workspace and assign it to the capacity
  • ☑ Create your first Lakehouse — it provisions a Delta table store and SQL endpoint automatically
  • ☑ Ingest data with a Data Factory pipeline or a Spark notebook
  • ☑ Build a Power BI report directly on top of the Lakehouse SQL endpoint

Key Takeaways

  • Microsoft Fabric unifies six analytics workloads on a single SaaS platform
  • OneLake is the shared storage layer — open Delta Parquet format, one namespace per tenant
  • All workloads share a single Capacity (CU pool), identity model, and governance layer
  • It replaces the need to integrate ADF + Synapse + Databricks + Power BI separately
  • Lakehouses are the primary storage pattern — tables exposed automatically via SQL endpoint
